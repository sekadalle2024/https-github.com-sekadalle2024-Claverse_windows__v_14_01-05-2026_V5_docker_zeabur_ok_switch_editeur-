feat: Integration frontend ACTIF avec colonnes BRUT, AMORT, NET

Résolution du problème d'affichage des colonnes BRUT, AMORT ET DEPREC, NET
dans le menu accordéon du bilan ACTIF.

## Problème
Le backend générait correctement le HTML enrichi avec les 7 colonnes, mais
le frontend l'ignorait et reconstruisait son propre HTML simplifié avec
seulement la colonne NET.

## Solution
Modification de public/menu.js pour utiliser le HTML généré par le backend:
- insertEtatsFinanciersResults(): Détection et utilisation de result.html
- setupEtatsFinanciersAccordions(): Support des accordéons backend

## Avantages
- Rétrocompatibilité: Fallback vers l'ancien format si result.html absent
- Séparation des responsabilités: Backend calcule, Frontend affiche
- Conformité SYSCOHADA: 7 colonnes conformes au format liasse officielle
- Maintenabilité: Code modulaire, tests automatisés, documentation complète
