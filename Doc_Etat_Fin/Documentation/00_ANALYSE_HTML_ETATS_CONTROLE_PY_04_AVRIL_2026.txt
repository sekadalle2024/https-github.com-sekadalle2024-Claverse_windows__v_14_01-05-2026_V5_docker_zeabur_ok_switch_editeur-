1. format_montant_controle(montant: float) -> str
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   DESCRIPTION:
   Formate un montant pour l'affichage dans les contrôles
   
   PARAMÈTRES:
   • montant (float): Montant à formater
   
   RETOUR:
   • str: Montant formaté avec espaces comme séparateurs de milliers
          ou "-" si le montant est quasi-nul (< 0.01)
   
   LOGIQUE:
   • Si |montant| < 0.01 → retourne "-"
   • Sinon → formate avec séparateurs d'espaces (ex: "181 162 530")
   
   EXEMPLES:
   format_montant_controle(181162530.0)  → "181 162 530"
   format_montant_controle(0.005)        → "-"
   format_montant_controle(-189540500.0) → "-189 540 500"


2. generate_etat_controle_html(etat_controle: Dict, section_id: str) -> str
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   DESCRIPTION:
   Génère le HTML pour un état de contrôle individuel (un accordéon)
   
   PARAMÈTRES:
   • etat_controle (Dict): Dictionnaire contenant:
       - titre (str): Titre de l'état de contrôle
       - postes (List[Dict]): Liste des postes avec:
           * ref (str): Référence du poste (ex: "CA", "PA")
           * libelle (str): Libellé du poste
           * montant_n (float): Montant exercice N
           * montant_n1 (float): Montant exercice N-1
   • section_id (str): Identifiant unique de la section
   
   RETOUR:
   • str: HTML complet de l'accordéon
   
   STRUCTURE HTML GÉNÉRÉE:
   <div class="etats-fin-section" data-section="{section_id}">
       <div class="section-header-ef">
           <span>🔍 {titre}</span>
           <span class="arrow">›</span>
       </div>
       <div class="section-content-ef">
           <table class="liasse-table">
               <thead>...</thead>
               <tbody>
                   <tr class="{row_class}">
                       <td class="ref-cell">{ref}</td>
                       <td class="libelle-cell">{libelle}</td>
                       <td class="montant-cell">{montant_n}</td>
                       <td class="montant-cell">{montant_n1}</td>
                   </tr>
               </tbody>
           </table>
       </div>
   </div>
   
   LOGIQUE SPÉCIALE:
   • Détecte les lignes de totaux (contenant "Total", "Équilibre", "Variation")
   • Applique la classe CSS "total-row" aux lignes de totaux
   • Retourne une chaîne vide si etat_controle est vide ou sans postes


3. generate_all_etats_controle_html(etats_controle: Dict) -> str
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   DESCRIPTION:
   Génère le HTML pour tous les états de contrôle dans un ordre défini
   
   PARAMÈTRES:
   • etats_controle (Dict[str, Dict]): Dictionnaire contenant tous les états
   
   RETOUR:
   • str: HTML complet de tous les accordéons concaténés
   
   ORDRE DE GÉNÉRATION:
   1. etat_controle_bilan_actif (Exercice N)
   2. etat_controle_bilan_actif_n1 (Exercice N-1)
   3. etat_controle_bilan_passif (Exercice N)
   4. etat_controle_bilan_passif_n1 (Exercice N-1)
   5. etat_controle_compte_resultat (Exercice N)
   6. etat_controle_compte_resultat_n1 (Exercice N-1)
   7. etat_controle_tft (Exercice N)
   8. etat_controle_tft_n1 (Exercice N-1)
   9. etat_controle_sens_comptes
   10. etat_equilibre_bilan
   
   LOGIQUE:
   • Parcourt l'ordre défini
   • Génère le HTML uniquement pour les états présents dans le dictionnaire
   • Concatène tous les HTML générés

STRATÉGIE D'INTÉGRATION:

OPTION 1: UTILISER LE HTML GÉNÉRÉ PAR PYTHON
• Le backend génère le HTML complet
• Le front-end affiche le HTML directement (dangerouslySetInnerHTML)
• Avantage: Pas de duplication de logique
• Inconvénient: Moins de contrôle côté React

OPTION 2: UTILISER LES DONNÉES JSON + COMPOSANT REACT
• Le backend retourne les données JSON
• Le composant React génère le HTML
• Avantage: Contrôle total côté React, interactivité
• Inconvénient: Duplication de la logique de formatage

RECOMMANDATION: OPTION 2 ✅
• Plus flexible
• Meilleure expérience utilisateur
• Facilite les futures évolutions
• Cohérent avec l'architecture React
