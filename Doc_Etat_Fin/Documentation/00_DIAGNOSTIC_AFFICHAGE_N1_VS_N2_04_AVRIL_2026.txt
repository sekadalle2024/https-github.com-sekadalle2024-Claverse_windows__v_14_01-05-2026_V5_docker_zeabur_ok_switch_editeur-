   CAUSE RACINE

🔍 ANALYSE :

Le problème n'est PAS un bug de calcul, mais une LIMITATION DE CONCEPTION :

1. Le format liasse officielle SYSCOHADA est conçu pour afficher 2 colonnes :
   - Exercice N
   - Exercice N-1

2. Pour afficher N-1 vs N-2, il faudrait :
   - Soit ajouter une 3ème colonne (N-2)
   - Soit créer un 2ème menu accordéon avec N-1 vs N-2

3. Le backend calcule bien les données, mais le frontend ne les affiche pas
